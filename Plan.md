# Plan - Cycle 5: 10번 프레임 보너스 투구 (확인 테스트)

## 목표
10번 프레임이 스페어/스트라이크로 끝나 보너스 투구(1~2회)가 추가될 때 `score()`가
올바른지 확인. 기존 인덱스 기반 순회(`rolls[i]`, `rolls[i+1]`, `rolls[i+2]`)가
"몇 번째 프레임인지"를 구분하지 않고 그냥 다음 투구를 읽는 방식이라, 보너스 투구도
이미 올바르게 처리될 가능성이 높음 — 이를 확인하는 사이클.

## 테스트
- 파일: `test_game.py`
- `test_tenth_frame_spare_with_bonus_roll`: 프레임1~9 거터(18회 0) + 10번 프레임 9,1(스페어) + 보너스 1회(5)
  - 기대 점수: 0*9 + (10+5) = 15
- `test_tenth_frame_strike_with_bonus_rolls`: 프레임1~9 거터(18회 0) + 10번 프레임 스트라이크(10) + 보너스 2회(4,5)
  - 기대 점수: 0*9 + (10+4+5) = 19

## 접근 방식
- 새 프로덕션 코드 작성 전, 기존 로직으로 통과하는지 먼저 실행
- 통과하면: 확인 테스트로 커밋 (일반화 완료 확인, 코드 변경 없음)
- 실패하면: 실패 원인 분석 후 Plan 갱신, 최소 코드로 수정
