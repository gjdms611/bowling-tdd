# CLAUDE.md

이 파일은 이 저장소 작업 시 Claude Code(claude.ai/code)에게 지침을 제공한다.

## 상태

빈 프로젝트 스캐폴드. `.venv`(Python 3.11.5, virtualenv, 패키지 미설치)만 존재 — 소스 파일, 테스트 파일, 의존성 매니페스트 아직 없음. Bowling Game Kata를 TDD로 새로 시작.

## 설정

무언가 설치하거나 테스트 실행 전 기존 venv 활성화:

```
.venv\Scripts\activate
```

테스트 프레임워크 아직 미설치. 기본 선택: `pytest` (`pip install pytest`, 실행은 `pytest`).

## The Kata

Bowling Game Kata(미국식 텐핀 볼링 점수 계산) 구현. 필수 클래스:

- `Game.roll(pins)` — 공 한 번 던질 때마다 호출; `pins`는 쓰러뜨린 핀 개수.
- `Game.score()` — 게임 총점 반환.

점수 규칙:
- **스페어**(한 프레임 두 번 던져 합 10): 프레임 점수 = 10 + *다음* 굴림의 핀 수.
- **스트라이크**(프레임 첫 굴림이 10): 프레임 즉시 종료; 프레임 점수 = 10 + *다음 두 번* 굴림의 핀 수.
- **10번째 프레임**: 스페어나 스트라이크면 프레임 완성을 위한 추가 보너스 굴림 1~2회 허용(10번째 프레임 최대 총 3굴림).

이 kata에서 명시적으로 범위 밖(검증 추가하지 말 것):
- 굴림의 핀 수가 유효한지 검증.
- 굴림/프레임 개수가 올바른지 검증.
- 프레임별/중간 점수 노출 — `score()`를 통한 최종 총점만.

## 개발 방식

테스트 먼저 작성(red-green-refactor): 시나리오 하나(오픈 프레임, 스페어, 스트라이크, 올스트라이크/"퍼펙트 게임", 10번째 프레임 보너스 굴림)에 대한 실패하는 테스트 작성 후 최소 코드로 통과시키고 다음 시나리오로 이동. 입력 검증이나 프레임 점수 리포팅 추가 금지 — 위에서 명시적으로 제외됨.
